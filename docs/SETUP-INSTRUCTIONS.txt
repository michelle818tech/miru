STEP 4: CONNECT ODOO (GIVE TO TECHMARBLES)

Your Odoo team needs to run this Python script WEEKLY (e.g., Sunday 5 AM):

```python
import requests
import json
from datetime import datetime

# Configuration
SUPABASE_URL = "https://your-supabase.supabase.co"
SYNC_ENDPOINT = f"{SUPABASE_URL}/api/sync-odoo"

# Get week number
week_num = datetime.now().isocalendar()[1]
year = datetime.now().year

# Extract from Odoo POS for each manager
managers_data = [
    {
        "email": "tsogzolmaa@mirus.mn",
        "sales": 1234567,  # From Odoo: SUM(pos_order.amount_total) WHERE cashier=manager
        "transactions": 156,  # Count of pos_order records
        "avg_value": 7908,  # sales / transactions
        "bundle_rate": 45.5,  # % of orders with 2+ items
        "refund_rate": 2.1,  # % of orders with refund_approval_status='approved'
        "top_products": ["Anua Cleanser", "Black Rouge", "Purito Sunscreen"]
    },
    # Add other 3 managers similarly...
]

for mgr in managers_data:
    response = requests.post(
        SYNC_ENDPOINT,
        json={
            "manager_email": mgr["email"],
            "week": week_num,
            "year": year,
            "sales_amount": mgr["sales"],
            "transaction_count": mgr["transactions"],
            "avg_transaction_value": mgr["avg_value"],
            "bundle_attach_rate": mgr["bundle_rate"],
            "refund_rate": mgr["refund_rate"],
            "top_products": mgr["top_products"],
        }
    )
    print(f"Synced {mgr['email']}: {response.status_code}")
```

Schedule this in Odoo Scheduler or cron job (weekly, e.g., Sundays 5 AM).

HOW IT WORKS

MANAGER WORKFLOW (Weekly):
1. Log in to website
2. See their Odoo metrics (auto-pulled):
   - Sales ₮ this week
   - # of transactions
   - Avg order value
   - Bundle rate
   - Refund rate
3. Write weekly reflection:
   - "Which products did you focus on?" (must be specific)
   - "What actions did you take?" (must have numbers/dates)
4. Submit → AI checks for vague/generic content
   - If APPROVED: ✓ saved
   - If REJECTED: ⚠️ shows errors ("too vague", "need product names", etc.)

MONTHLY (Last week of month):
- Manager submits WHAT % and HOW %
- HR reviews & approves
- System calculates KPI tier:
  • 90-100% overall = 100% KPI ✓
  • 80-89% = 70% KPI
  • 70-79% = 40% KPI
  • <70% = 0% KPI

HR WORKFLOW:
1. Dashboard shows all 4 managers
2. Click each manager to see:
   - Weekly logs (with AI feedback)
   - Odoo metrics (sales trend, quality indicators)
   - Monthly scores
3. HR can write:
   - Work Ethics notes (behavior, cursing, attendance)
   - Warnings section (things to improve)
4. Approve monthly submissions

QUALITY ASSESSMENT (AI Blocks Vague Text)

BAD (gets blocked):
✗ "Gave content manager a list of slow-moving products"
✗ "Increased sales"
✗ "Worked hard on merchandise"
✗ "Improved customer service"

GOOD (gets approved):
✓ "Focused on Anua Cleanser (45 units sold, +23% vs week 1) and Black Rouge (67 units, +18%). These were Q2 slow movers. Created end-cap displays Monday-Wednesday, trained staff on benefits."
✓ "Ran 20% discount July 8-10, moved 3 units to prime shelf, baseline sales held +15% after promo ended"

The system checks for:
- Specific product names
- Numbers (amounts, percentages, units)
- Dates/timeframes
- Concrete actions (not just "worked on")
